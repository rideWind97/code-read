# React 为什么要有合成事件

React 设计合成事件（SyntheticEvent）主要有以下原因：

1. 兼容性统一
- 不同浏览器原生事件存在差异，React 用合成事件封装后，保证事件行为和属性在各浏览器下表现一致。

2. 性能优化
- React 通过事件委托，把所有事件统一绑定到根节点（如 document），减少事件监听器数量，提升性能。

3. 跨平台支持
- 合成事件不仅能在浏览器端用，也方便未来扩展到原生、服务端等其他平台。

4. 便于管理
- 合成事件生命周期由 React 控制，便于事件回收和批量更新。

## 总结
合成事件让 React 事件系统更高效、更安全、更易维护，屏蔽了底层差异，提升了开发体验。
